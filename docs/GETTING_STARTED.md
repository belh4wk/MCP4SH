# MCP4SH® ::: Getting Started

This is the shortest sensible path to a first drive.

## 1) Install carefully

- Download the release from the official GitHub release page.
- Verify the SHA-256 checksum first.
- Close SimHub.
- Run the installer.
- Start SimHub and enable **MCP4SH** in Add/remove features if needed.

Read:

- `SECURITY.md`
- `INSTALLER_SECURITY.md`

## 2) Run Setup Assistant

Use Setup Assistant before you start tuning.

It helps you:

- select the shakers physically installed on your rig
- send test pulses to each output channel
- tell MCP4SH what you actually felt
- generate a matching SimHub sound output profile
- open the profile folder
- follow the SimHub Helper steps to import the profile
- use the pulse visual and hover routing readout to sanity-check mapped shakers

Generated profiles are written to:

```text
Documents\SimHub\MCP4SH
```

## 3) Import the supplied effect profile

In SimHub:

1. Open **ShakeIt Bass Shakers**.
2. Import the supplied MCP4SH Standard effects profile from `Documents\SimHub\MCP4SH`.
3. Import/apply the sound output profile generated by Setup Assistant.
4. Check that each channel hits the shaker you expect.

Do not assume any profile is perfectly tuned for your rig before checking the physical routing.

## 4) Set gains in the right order

This is the bit people get wrong.

1. Confirm routing with Setup Assistant.
2. Tune **MCP4SH plugin gains** first via the GUI.
3. Tune **SimHub master/global output** second.
4. Touch **SimHub ShakeIt Bass Shaker Effect Profile per-effect gains** last, only if needed.

## 5) First test session

Use one sim and one car you know well.

Look for:

- clean front vs rear timing
- brake cues that make sense under your foot
- engine / drivetrain presence without drowning everything else
- tyre scrub that feels informative rather than random
- suspension vibration that feels like surface/body texture, not constant impact spam

## 6) If it feels wrong

Do not immediately nuke every slider.

Instead:

- verify routing first
- lower gains a bit
- check `docs/KNOWN_ISSUES.md`
- enable logging for a short test if needed

See `docs/LOGGING.md`.


## Updated bundled profiles

If the plugin shows an updated profile notice, choose **Show details...** to open Setup Assistant directly to **SimHub Helper → Updates**. That page shows the local profile filename and the SimHub import steps.

MCP4SH does not automatically activate or overwrite your SimHub profiles. Import the refreshed Standard profile manually when you want to use it.
